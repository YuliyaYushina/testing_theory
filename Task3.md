# **План тестирования**

## **Проект: Задание 3: тестирование API: Расчет стоимости продукта**

## **Версия: <1.0>**

### **1. Введение**

#### **1.1. Цель**

Данный документ описывает стратегию тестирования реализованного REST API GET /products/{productId}/cost?authToken=&count=&start= 

**Цель** - проверка корректности алгоритма циклического расчета стоимости продукта, работы системы авторизации и обработки ошибок.

#### **1.2. Обоснование**

Метод REST API GET /products/{productId}/cost?authToken=&count=&start= позволяет запрашивать стоимость продукта. В качестве входных данных метод принимает: 
- count - количество экземпляров продукта, за которое нужно получить стоимость,
- start - индекс с которого необходимо вычислять стоимость (количество уже приобретенных продуктов).

#### **1.3. Область тестирования**

**Этапы тестирования:** функциональное тестирование API, тестирование безопасности, негативное тестирование.

**Что тестируем:** расчет стоимости продукта, авторизация, обработка ошибок.

**Что НЕ тестируем:** UI-тестирование, тестирование производительности (Load/Stress).

**Риски:** математические ошибки в алгоритме циклического расчета, обход авторизации.

### **2. Требования к тестированию**

- Проверка корректности авторизации (16-значный authToken).
- Проверка корректности алгоритма циклического расчета стоимости продукта.
- Проверка кодов ответа HTTP: 200, 401, 404, 500.
- Валидация структуры и содержания JSON-ответа для позитивных сценариев (код 200).
- Валидация структуры и содержания JSON-ответа для негативных сценариев (код 500).

### **3. Стратегии тестирования**

#### **3.1. Типы тестирования**

##### **3.1.2. Функциональное тестирование**

**Объект:** проверка корректности алгоритма циклического расчета стоимости продукта.

**Техника:** Таблица принятия решений, эквивалентное разделение и анализ граничных значений. Позитивные и негативные сценарии (невалидный тип данных, отрицательные значения, нулевое количество, дробное число).

**Критерий завершения:** 100% критических сценариев пройдены успешно.


##### **3.1.9. Тестирование безопасности и контроля доступа**

**Объект:** Проверка authToken - набор из 16 символов, включающие цифры и буквы. 

**Техника:** Граничные значения для authToken (16 символов (цифры+буквы), 15 символов, 17 символов, пусто, спецсимволы) 

**Критерий завершения:** при корректном токене - получение стоимости продукта, при неверном или отсутствующем токене - ответ с кодом 401.
